# SQL Injection Vulnerability Analysis Lab

## Platform
PortSwigger Web Security Academy

## Objective
Identify and exploit a SQL injection vulnerability in a controlled web application environment.

## Vulnerability
The application was vulnerable through the product category filter parameter.

## Testing Process
- Modified URL parameters
- Tested input handling
- Used boolean-based SQL injection logic
- Observed unauthorized data retrieval

## Impact
An attacker could manipulate database queries and access information that was not intended to be displayed.

## Remediation
- Use parameterized queries
- Validate user input
- Implement secure coding practices
